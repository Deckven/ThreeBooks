# ThreeBooks — Инструкции для Claude Code

## О проекте

Три взаимосвязанные нон-фикшн книги на русском языке:

1. **«Контуры расколотого мира»** (`book1-fractured-world/`) — футурология, ближайшие 50–70 лет человечества. Геополитика, технологии, демография, климат, конфликты цивилизаций.

2. **«Пятая информационная революция»** (`book2-fifth-revolution/`) — следующий скачок в информационных технологиях. ИИ, квантовые вычисления, нейроинтерфейсы, трансформация общества.

3. **«Поколение двойного шторма»** (`book3-double-storm/`) — украинское поколение X, 35 лет независимости, «что пошло не так», война с Россией. Личный опыт, свидетельства, анализ.

Книги пересекаются тематически (технологии, геополитика, поколенческий опыт). Общие ресурсы хранятся в `_shared/`.

## Языковые конвенции

- **Содержимое файлов**: русский язык
- **Имена файлов и директорий**: английский язык
- **Теги и статусы**: английский язык (draft, supported, refuted и т.д.)

## Структура проекта

```
_shared/          — общие ресурсы (источники, концепции, таймлайны, перекрёстные ссылки)
_templates/       — шаблоны для создания новых файлов
book1-fractured-world/   — Книга 1
book2-fifth-revolution/  — Книга 2
book3-double-storm/      — Книга 3
```

Каждая книга содержит: `book.md` (метаданные), `outline.md` (структура), `research/` (исследования), `drafts/` (черновики), `notes/` (заметки), `sources/` (источники).

## Рабочие команды

### Исследование темы
Запрос: `research [тема] for [book1/book2/book3]`
Действия:
1. Создать файл в `book*/research/topics/` по шаблону `_templates/topic-research.md`
2. Провести исследование, заполнить карточку
3. Обновить `_shared/sources/sources.md` при добавлении новых источников
4. Проверить перекрёстные ссылки с другими книгами

### Гипотеза
Запрос: `hypothesis [утверждение] for [book1/book2/book3]`
Действия:
1. Создать файл в `book*/research/hypotheses/` по шаблону `_templates/hypothesis.md`
2. Проанализировать аргументы за и против
3. Обновить статус: `draft → under_review → supported / refuted / revised`

### Проверка факта
Запрос: `check [утверждение] for [book1/book2/book3]`
Действия:
1. Создать файл в `book*/research/fact-checks/` по шаблону `_templates/fact-check.md`
2. Проверить по доступным источникам
3. Выставить вердикт: `confirmed / partially_confirmed / refuted / unverifiable`

### Черновик главы
Запрос: `draft [глава] for [book1/book2/book3]`
Действия:
1. Создать/обновить файл в `book*/drafts/` по шаблону `_templates/chapter-draft.md`
2. Опираться на `outline.md`, исследования и заметки

### Перекрёстные ссылки
Запрос: `link [концепция] across books`
Действия:
1. Обновить `_shared/cross-references.md`
2. Обновить `_shared/concepts/index.md` при необходимости

### Сценарий (только book1)
Запрос: `scenario [название] for book1`
Действия:
1. Создать файл в `book1-fractured-world/research/scenarios/` по шаблону `_templates/scenario.md`

## Стандарты работы с источниками

- Каждый источник получает оценку надёжности: `high / medium / low / unverified`
- Критерии: авторитетность автора, рецензирование, подтверждение другими источниками
- Все источники регистрируются в `_shared/sources/sources.md` или `book*/sources/`
- Формат записи — по шаблону `_templates/source-entry.md`

## Протокол обсуждения гипотез

1. **Формулировка** — чёткий тезис
2. **Аргументы «за»** — с указанием источников
3. **Аргументы «против»** — с указанием источников
4. **Альтернативные интерпретации** — другие объяснения тех же данных
5. **Слепые зоны** — что мы можем упускать?
6. **Вывод** — поддержана / опровергнута / требует доработки
